# Dynamic General Analysis Protocol

## Overview

Execute comprehensive analysis of any files or directories using adaptive reconnaissance and intelligent pattern detection. This command dynamically determines the analysis approach based on the discovered content type, making it universally applicable to any codebase, configuration, or documentation.

## Mission Briefing: Dynamic General Analysis Protocol

**ANALYSIS COMMAND DETECTION:** Before proceeding with analysis, check if the provided text contains any `/analyze*` command (e.g., `/analyze-code`, `/analyze-project`, `/analyze-bug-*`, etc.). If found, skip this command's analysis and execute the detected command instead to avoid duplicate analysis.

You will now execute a comprehensive analysis of any specified targets using the **AUTONOMOUS PRINCIPAL ENGINEER - OPERATIONAL DOCTRINE.** This analysis follows the Phase 0 reconnaissance principles with intelligent adaptation based on discovered content types. The goal is to map and understand the system architecture, patterns, and purpose, not to provide recommendations or improvements.

---

## **Phase 0: Dynamic Content Discovery & Analysis Strategy (Read-Only)**

**Directive:** Perform intelligent reconnaissance to discover content types and adapt analysis strategy accordingly.

**Content Discovery Scope:**

- **File Type Detection**: Identify file extensions, formats, and content patterns
- **Purpose Recognition**: Determine if analyzing code, configuration, documentation, or mixed content
- **Technology Stack Detection**: Discover frameworks, tools, and technologies in use
- **Architecture Pattern Recognition**: Identify architectural patterns and organizational structures
- **Analysis Strategy Selection**: Choose appropriate analysis depth and methodology based on content

**Constraints:**

- **No mutations are permitted during this phase**
- **No recommendations or improvement suggestions should be provided**
- **Focus solely on understanding and mapping the discovered content**

---

## **Phase 1: Intelligent Content Classification**

**Directive:** Dynamically classify discovered content and select appropriate analysis strategies.

**Classification Categories:**

1. **Code Analysis**:
   - **Frontend Code**: Angular, Vue, Svelte, or vanilla JS/TS
   - **Backend Code**: Node.js, Python, Java, C#, Go, or other server-side languages
   - **Configuration Code**: Build tools, CI/CD, deployment scripts
   - **Database Code**: SQL, NoSQL schemas, migrations
   - **Infrastructure Code**: Docker, Kubernetes, Terraform, CloudFormation

2. **Configuration Analysis**:
   - **Build Configuration**: Angular CLI, Webpack, Vite, Rollup, esbuild, or other bundlers
   - **Package Management**: package.json, requirements.txt, Cargo.toml, etc.
   - **Environment Configuration**: .env files, config files, environment variables
   - **Tool Configuration**: ESLint, Prettier, TypeScript, testing frameworks
   - **Deployment Configuration**: Docker, Kubernetes, CI/CD pipelines

3. **Documentation Analysis**:
   - **Technical Documentation**: README, API docs, architecture docs
   - **Code Documentation**: Comments, JSDoc, docstrings, inline docs
   - **Process Documentation**: Workflows, procedures, guidelines
   - **Design Documentation**: Wireframes, mockups, design systems
   - **Cursor Rules & Commands**: Cursor-specific rules, commands, and operational doctrines

4. **Data Analysis**:
   - **Structured Data**: JSON, XML, YAML, CSV, database dumps
   - **Log Files**: Application logs, error logs, access logs
   - **Test Data**: Fixtures, mock data, test cases
   - **Schema Data**: Database schemas, API schemas, data models

**Adaptive Analysis Strategy Selection:**
- **Code Files**: Apply code analysis methodology with framework-specific adaptations
- **Configuration Files**: Focus on configuration patterns, dependencies, and system setup
- **Documentation Files**: Analyze information architecture, completeness, and clarity
- **Mixed Content**: Apply multi-layered analysis covering all discovered content types

**Technology Stack Categories:**
